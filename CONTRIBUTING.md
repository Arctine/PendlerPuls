# Contributing

PendlerPuls is currently a one-student portfolio project, but changes should
still follow a small, repeatable workflow.

1. Create a short branch with one purpose.
2. Keep code and documentation in the same change when behavior changes.
3. Run the backend tests, frontend tests, and production frontend build.
4. Do not commit generated output, local databases, logs, or secrets.
5. Explain limitations instead of hiding unfinished behavior.

Commit messages should describe the result, for example:

```text
Add saved journey ownership tests
Document PostgreSQL deployment
Handle Entur timeout responses
```
